# REQUISITOS DO SISTEMA
## Hospital Agostinho Neto - Plataforma de Marcação de Consultas

---

## 1. CONTEXTO DO NEGÓCIO

O Hospital Agostinho Neto pretende reduzir filas e melhorar o processo de marcação de consultas através de uma plataforma digital.

### Problemas atuais:
- Sobreposição de horários
- Conflitos de agenda
- Longos tempos de espera
- Falta de prioridade para pacientes urgentes

---

## 2. OBJETIVO DO PROJETO

Garantir uma gestão eficiente, segura e confiável das marcações médicas, reduzindo conflitos de agenda e assegurando prioridade adequada aos casos urgentes.

---

## 3. REQUISITOS FUNCIONAIS

| # | Requisito | Descrição |
|---|-----------|-----------|
| RF01 | Marcação de Consultas | Paciente escolhe especialidade, médico, data e hora disponível |
| RF02 | Reagendamento | Paciente altera data e hora da consulta |
| RF03 | Cancelamento | Paciente desmarca a consulta |
| RF04 | Gestão de Médicos | Administrador controla horários e especialidades |
| RF05 | Gestão de Prioridades | Sistema define atendimento prioritário para casos urgentes |
| RF06 | Triagem Automática | Classificação inicial com base nos sintomas informados |
| RF07 | Prioridade Clínica | Atribuição automática (Urgente, Alta, Médio ou Baixa) |
| RF08 | Notificações | Envio de confirmações e lembretes aos pacientes |

---

## 4. REQUISITOS NÃO FUNCIONAIS

| # | Requisito | Descrição |
|---|-----------|-----------|
| RNF01 | Fiabilidade | Sistema não pode perder dados, atualiza horários em tempo real |
| RNF02 | Segurança | Dados clínicos protegidos e criptografados |
| RNF03 | Usabilidade | Interface simples e intuitiva |
| RNF04 | Desempenho | Tempo de resposta inferior a 500ms |
| RNF05 | Disponibilidade | Sistema disponível 99,9% do tempo |
| RNF06 | Concorrência | Suporte para múltiplos usuários simultâneos |

---

## 5. MUDANÇA DE REQUISITOS

Após reunião com a direção clínica, o sistema deverá implementar:

- **Triagem automática:** Classificação inicial com base nos sintomas
- **Prioridade clínica:** Atribuição automática (Urgente, Alta, Médio ou Baixa)

---

## 6. QUALIDADE QUE SALVA VIDA

**Tecnologia, Qualidade e Empatia** para um atendimento melhor.
